# DiffuFont 按 AnyText2 思路的 S1 改造方案 + 低分辨率 AttnX（两处）改造文档（速度优先，单字 glyph）

> 目标：在 **不牺牲结构正确性** 的前提下，优先提升推理速度。  
> 路线：先做 S1（缓存条件特征 + 拔掉最贵的 DACA 层），再加 **两个低分辨率 AttnX** 用于在减少条件计算后保持风格一致性。

---

## 0. 你当前代码的“插层现状”（作为改造基线）

### DACA 的插入点（现状）
- 位于 `DownBlock.forward()`：`conv1(stride=2)` 之后立刻调用 `self.daca(x, content_feat, t)`  
- 最后一个 down block 默认不启用

### FGSA 的插入点（现状）
- `DecoderResBlock`（bottleneck 低分辨率）里使用 FGSA
- `UpBlock.forward()` 里 `conv1` 后使用 FGSA + AdaLN
- 最后一个 up block 默认不启用 FGSA/AdaLN（已经是正确的省算力策略）

### 最大性能问题（现状）
- 在 `FontDiffusionUNet.forward()` 内每次都重算：
  - `content_encoder(content_img)`
  - `style_encoder(style_img)`
- 采样（DDIM/PLMS）每个 timestep 都会调用一次 forward → encoder 重复计算 N 次（N=采样步数）

## 1. S1 配置表（最终想达到的开关状态）

> 输入为 256×256 单字 glyph，默认 `num_layers=4` 的典型分辨率路线：

- Down path: 128 → 64 → 32 → 16
- Up path:   16 → 32 → 64 → 128 → 256

### 1.1 DACA 开关（S1）
| Block | 分辨率 | 开关 |
|---|---:|---|
| DACA@Down[0] | 128×128 | ❌ 关闭 |
| DACA@Down[1] | 64×64 | ✅ 保留 |
| DACA@Down[2] | 32×32 | ✅ 保留 |
| DACA@Down[3] | 16×16 | -（原本不启用） |

> 解释：DACA 在 128×128 最贵（grid_sample + R 次循环），关掉这一层通常速度收益最大、结构损失最小。

### 1.2 FGSA 开关（S1）
| Block | 分辨率 | 开关 |
|---|---:|---|
| FGSA@Bottleneck(DecoderResBlock) | 16×16 | ✅ 保留 |
| FGSA@Up[0] | 16→32 | ✅ 保留 |
| FGSA@Up[1] | 32→64 | ✅ 保留 |
| FGSA@Up[2] | 64→128 | ✅ 保留（如需更快可选关） |
| FGSA@Up[3] | 128→256 | -（原本已关闭） |

### 1.3 条件特征缓存（S1）
- `condition_cache = true`：采样循环外 `encode_conditions()` 只执行一次
- 采样循环内 `forward_with_feats()` 复用 features

## 2. S1 必改：条件特征预计算与复用（WriteNet 思想最小落地）

### 2.1 修改点 A：在 `models/font_diffusion_unet.py` 添加两个接口

#### A1) `encode_conditions(content_img, style_img)`
**输入：**
- `content_img`: (B, 1, 256, 256) 或 (B, C, H, W)（以你实现为准）
- `style_img`:   (B, K, 256, 256) 或拼接后的张量（以你实现为准）

**输出：**
- `content_feats`: list/tuple，多尺度特征（必须与原 DACA/UNet消费方式一致）
- `style_feats`:   list/tuple，多尺度特征（必须与 FGSA消费方式一致）

> 注意：不要改 encoder 的结构与输出格式，只是把它们从 forward 中拆出去。

#### A2) `forward_with_feats(x_t, t, content_feats, style_feats)`
**输入：**
- `x_t`: 当前扩散 latent（你的实现里通常是 (B, C, 256, 256)）
- `t`:   当前时间步（embedding 或标量）
- `content_feats/style_feats`: 来自 `encode_conditions` 的缓存特征

**输出：**
- 与原 `forward()` 一致（比如预测噪声或 x0）

#### A3) 保留原 `forward(x_t, t, content_img, style_img)` 兼容
实现方式建议：
- `forward()` 内部仅做：
  - `content_feats, style_feats = encode_conditions(...)`
  - `return forward_with_feats(x_t, t, content_feats, style_feats)`

这样训练/推理旧调用都不崩。

---

### 2.2 修改点 B：在 `models/model.py` 的 `ddim_sample()`（或采样函数）挪出 encoder

**改造前（现状逻辑）**
- for each timestep:
  - `pred = model(x_t, t, content_img, style_img)`

**改造后（S1）**
- loop 外：
  - `content_feats, style_feats = model.encode_conditions(content_img, style_img)`
- loop 内：
  - `pred = model.forward_with_feats(x_t, t, content_feats, style_feats)`

> 这一步对速度提升最大，通常不会影响质量（数学等价，只是避免重复计算）。

---

## 3. S1 推荐改：按层开关 DACA（拔掉 Down[0]）

### 3.1 修改点 C：DownBlocks 支持 per-layer 的 `use_daca`
在构建 `down_blocks` 的地方新增配置：
- `daca_layers = [False, True, True, False]`

创建每个 DownBlock 时传入 `use_daca=daca_layers[i]`

在 `DownBlock.forward()` 内：
- `if self.use_daca: x = self.daca(x, content_feat, t)`

> 必须保持 `content_feat` 的尺度与该层特征一致（通常你已有对应的 `content_feats[i]`）。

## 4. S1 + 两个低分辨率 AttnX：用于“稳风格、少漂移”

> 你现在 UNet 不是典型 Transformer UNet，所以建议做 **轻量 AttnX-like 融合块**：  
> “用 attention/门控把 style 特征按位置吸收到 U-Net 特征里”，并采用残差形式避免破坏结构。

### 4.1 AttnX 插入位置（只插 2 处，速度友好）

#### AttnX #1：Bottleneck（16×16）
- 插在 `DecoderResBlock` 之前或之后（推荐：之后）
- 理由：最小分辨率，attention 计算最便宜；影响“整体风格一致性”最大。

#### AttnX #2：Up[0]（16→32 的第一个上采样块）
- 插在 `UpBlock` 的 `conv1` 之后、FGSA 之前（或 FGSA 之后，二选一）
- 推荐顺序：`conv1 -> AttnX -> FGSA -> AdaLN -> upsample`
- 理由：仍是低分辨率，能影响笔画形态生成阶段，且开销可控。

> 强烈不建议在 64×64 以上加 AttnX（速度会线性下降且很明显）。

---

### 4.2 AttnX 的输入选取（非常具体）

为了与你现有模块兼容并尽量省算：
- **Query**：来自 U-Net 当前层特征 `h`（形状 B×C×H×W）
- **Key/Value**：来自 `style_feats` 的同尺度特征（或 FGSA 聚合后的 style 特征）
  - 例如 bottleneck 用 `style_feats_bottleneck`
  - Up[0] 用 `style_feats_16or32`（与注入点分辨率匹配）

如果你的 `style_feats` 是 list（按尺度排列），就直接取对应 index。

---

### 4.3 AttnX 形式（建议的“轻量实现”）

推荐两种实现方式（从更快到更强）：

#### 方案 X1（更快）：门控残差融合（无显式注意力）
- 先用 1×1 conv 投影 `style_feat` 到 `h` 的通道数
- 计算一个 gate（sigmoid）控制注入强度
- 残差相加：

`h = h + gate(t) * proj(style_feat)`

其中 `gate(t)` 可以是：
- t-embedding 过一个 MLP 输出标量/通道门控
- 或直接复用你 DACA 的 `alpha(t)` 形式（早期强/后期弱）

> 优点：最快，最不容易拖慢推理。

#### 方案 X2（更强，但仍可控）：低分辨率 Cross-Attn（只在 16×16 与 32×32 做）
- 将 `h` 展平为 tokens：`(B, HW, Cq)`
- 将 `style_feat` 展平为 `(B, HW, Ck)`
- 做一次 cross-attn：`Attn(Q=h, K=style, V=style)`
- 输出 reshape 回 `B×C×H×W`
- 用残差加回：

`h = h + alpha(t) * attn_out`

> 优点：比纯门控更“按位置选择性吸收风格”，更像 AnyText2 的 AttnX精神。  
> 注意：只在 16/32 做，不要扩展到高分辨率。

---

### 4.4 AttnX 的强度调度（避免破坏结构的关键）
建议使用 **随时间步变化** 的强度（t-dependent），原则：
- 高噪声阶段（结构建立）：AttnX 强度低或中等（避免把结构搞乱）
- 低噪声阶段（细节/笔触）：AttnX 强度更高（补风格细节）

简化版本：
- `alpha(t)` 线性/余弦从小到大（或反过来，视你实验）
- 建议先复用你 DACA 已有的 `alpha(t)` 结构（减少新超参）

## 5. “最不能改”的部分（字体生成任务底线）

### 5.1 不要破坏内容结构路径（content preservation）
- 字体任务最重要的是：字不能写错、部件不能丢
- 因此 **至少保留 64×64 和 32×32 的 DACA**（S1 已满足）
- 不要把所有 DACA 作为最终配置关掉（会导致复杂字结构崩）

### 5.2 不要改变训练数据的“同字体配对逻辑”
- 你训练时 content 与 style 必须来自同一字体（或有效的参考集合），否则风格学习会崩
- 参考字选择/配对（尤其中文部件映射思想）尽量保持不动

### 5.3 不要把损失函数主干换成“纯生成美学”
- 字体生成依赖重建/结构相关损失保证字形正确
- 可以加辅助 loss，但不要丢掉能约束字形结构的主损失

### 5.4 不要在高分辨率加重 attention
- 64×64 以上加入 cross-attn 会显著拖慢，违背你的“速度优先”
- AttnX 只放 16×16 和 32×32 两处即可

## 6. 验收/对比实验建议（保证你知道改造是否成功）

### 6.1 速度
- 固定采样步数（推荐先测 20、50）
- 比较改造前后 wall-clock time / 单字推理耗时

### 6.2 结构正确性（底线）
- 选复杂字（多部件/多笔画）做对比
- 观察：是否缺笔画、部件位置漂、结构被风格吞噬
- 若出现：降低 AttnX 强度或把 AttnX #2 移到 FGSA 之后

### 6.3 风格一致性
- 固定同一字体参考字集合，生成 20 个未见字
- 观察：粗细、端点装饰、弯钩风格是否一致
- 若漂移：提高 bottleneck AttnX 强度，或保留 FGSA@Up[2]

## 7. 一页式“你最终要做的改动清单”（最实用）

### 必做（S1）
- [ ] 在 UNet 中新增 `encode_conditions()` 与 `forward_with_feats()`
- [ ] 在采样循环中：循环外 encode，循环内复用 feats
- [ ] 配置 DACA：关 Down[0]，留 Down[1]/Down[2]

### 加两个低分辨率 AttnX（S1+AttnX）
- [ ] 在 bottleneck（16×16）插 AttnX
- [ ] 在 Up[0]（16→32）插 AttnX
- [ ] AttnX 使用残差 + alpha(t) 门控
- [ ] 严禁把 AttnX 扩展到 ≥64×64 的层
