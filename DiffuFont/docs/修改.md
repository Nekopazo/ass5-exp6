# 最小改动版：只改 3 个点

## 1. 主训练先删掉 `recon_l1` 和 `recon_perceptual`

你现在 Stage C 损失是：

$$L = \lambda_{diff} \cdot diffusion\_mse + \lambda_{rec} \cdot recon\_l1 + \lambda_{perc} \cdot recon\_perceptual$$

这是我最建议先动的。

### 改成

$$L = diffusion\_mse$$

先只用这个，别加别的。

### 为什么这是“最小改动里收益最大”的一条

因为你现在已经有：

* 预训练好的 VAE
* 预训练好的 style encoder
* latent diffusion 主干

这时候再强行加像素/感知重建，很容易把模型往“平均正确”拉，而不是往“条件生成正确”拉。

### 什么时候再考虑加回辅助项

只有在你确认：

* 纯 diffusion 下字形边缘明显崩
* 而且 sample 不是 style 不准，是纯视觉质量差

这时再尝试加一个**很小权重**的边缘项，不是先加 perceptual。

---

## 2. 不再“整支冻结 style”，改成只解冻最后一小部分

你当前 Stage C 默认冻结：

* `VAE`
* `style encoder` 整支。

VAE 继续冻，没问题。  
但 style 整支全冻，我建议做最小修改：

### 改成

#### 前半段训练

冻结：

* `VAE`
* `style_encoder`
* `global_style_pool`
* `local_style_pool`
* `style_global_proj`

只训练：

* `content_encoder`
* `content_proj`
* `diffusion backbone`

#### 后半段训练

继续冻结：

* `VAE`
* `style_encoder` 主体前几层

只解冻：

* `global_style_pool`
* `local_style_pool`
* `style_global_proj`
* 如果代码里方便，再加 `style_encoder` 最后一层

### 为什么这样就够了

你不想大改，那就别碰 style backbone 的大结构。  
真正最值得放开的，其实是**多参考聚合层**和**最后的 style 投影层**，因为它们最接近生成任务。

也就是说，先不改“style 特征提取”，只改“这些 style 特征怎么喂给生成器”。

### 学习率

解冻的这部分用主干的 `0.1x` 学习率就行。

---

## 3. 给 style cross-attention 加一个 gate

你现在每层 DiT block 都是：

1. self-attention
2. content cross-attention
3. style cross-attention
4. MLP  
    而且 `style_global` 还会进 AdaLN。
    
    当前模型架构
    

这个结构不用改。  
你只要加一个极小的改动：

### 在 style cross-attention 输出前乘一个可学习 gate

形式上就是：

$$h = h + g_l \cdot Attn_{style}(h, style\_tokens)$$

其中：

* `g_l` 是每层一个可学习标量
* 初始化成小值，比如 `0.1`

### 好处

这会解决两个常见问题：

* style 条件太强，参考字内容噪声带进来
* style 条件太弱，模型直接无视 style

而且这几乎不改整体结构，只是在 residual 上加个门。

### 这是最小改动里非常值得做的一项

因为你现在 style token 固定就是 `B x 128 x 512`，而且每层都读。  
不加 gate，模型对 style 的利用强弱完全不可控。
