# 当前模型架构

本文档描述的是 `2026-03-18` 之后这版 `content + style latent diffusion` 的当前实现。

## 1. 总览

工程仍保留三阶段训练：

1. `Stage A: VAE 预训练`
2. `Stage B: Style 预训练`
3. `Stage C: Diffusion 主训练`

当前主路径职责如下：

- `VAE`：把 `1 x 128 x 128` 灰度字形压缩到 `4 x 16 x 16` latent
- `Content encoder`：把内容字形编码成 `256` 个内容 token
- `Style encoder`：把 `R=8` 张参考字形编码成全局 token 和局部 patch token
- `Global + Static Local Style Pool`：输出 `style_global` 和更长的 `style_tokens`
- `Diffusion Transformer`：在 latent 空间做条件去噪生成

当前 `train.py` 默认配置：

- `device = cuda:1`
- `batch = 32`
- `style_ref_count = 8`
- `image_size = 128`
- `latent_channels = 4`
- `latent_size = 16`
- `encoder_hidden_dim = 512`
- `encoder_depth = 4`
- `dit_hidden_dim = 512`
- `dit_depth = 12`
- `local_style_tokens_per_ref = 16`
- `content_cross_attn_layers = 8`
- `style_cross_attn_every_n_layers = 1`
- `lr = 1e-4`

## 2. 数据与 Batch

数据由 `FontImageDataset` 提供：

- 按字体划分训练/验证，不按字符划分
- 每个样本包含 `content glyph`、`target glyph`、`style refs`、`style_ref_mask`
- 默认每个样本采样 `8` 张同字体不同字符的参考图

只有以下场景会额外构造正样本风格对：

- `Stage B: Style 预训练`
- `Stage C` 且 `diff_lambda_ctr > 0`

此时 batch 还会包含：

- `style_img_pos`
- `style_ref_mask_pos`

## 3. 模型结构

### 3.1 Glyph VAE

- 输入：`target glyph: B x 1 x 128 x 128`
- 编码后 latent：`z in R^(B x 4 x 16 x 16)`
- 解码器把 latent 还原回 `128 x 128` 灰度字形

### 3.2 Content Encoder

- patch size = `8`
- 每张图切成 `16 x 16 = 256` 个 patch
- `4` 层 transformer encoder
- 不使用 `cls token`

输出：

- `content_features in R^(B x 256 x 512)`
- `content_tokens in R^(B x 256 x 512)`

当前默认 `encoder_hidden_dim == dit_hidden_dim == 512`，因此 `content_proj` 实际是 `Identity`。

### 3.3 Style Encoder

输入：

- `style refs: B x R x 1 x 128 x 128`
- 默认 `R = 8`

单张参考图编码：

- patch size = `8`
- 每张图切成 `256` 个 patch
- patch embed 到 `512` 维
- 加 2D 位置编码
- 加 `cls token`
- `4` 层 transformer encoder

输出：

- `local_tokens in R^(B x R x 256 x 512)`
- `global_tokens in R^(B x R x 512)`

### 3.4 Global + Static Local Style 聚合

当前风格聚合固定为两条路径：

- `global_style_pool`
  - 对 `R` 个全局 token 做 attention pooling
  - 输出 `style_global in R^(B x 512)`
- `local_style_pool`
  - 不输入 `content`
  - 对每张参考图的 `256` 个 local patch token 做静态 attention pooling
  - 每张参考图默认输出 `16` 个 local style token

最终输出：

- `style_tokens in R^(B x (16R) x 512)`
- `style_global in R^(B x 512)`
- `style_token_mask in R^(B x 16R)`

默认 `R = 8` 时：

- `style_tokens in R^(B x 128 x 512)`
- `style_token_mask in R^(B x 128)`

这版实现里：

- `style_proj` 负责把 local style token 投影到 DiT hidden dim
- `style_global_proj` 负责把全局风格向量投影到 DiT hidden dim

### 3.5 Style Contrastive Head

只在 `Stage B` 或 `Stage C` 显式启用对比损失时使用。

当前对比头仍挂在 `style_global` 上：

- `Linear(512 -> 512)`
- `GELU`
- `Linear(512 -> 128)`
- `L2 normalize`

输出：

- `z_style in R^(B x 128)`

## 4. Diffusion Transformer Backbone

工作空间：

- latent 空间
- 输入 latent 为 `B x 4 x 16 x 16`

条件输入保持不变：

- `content_tokens`
- `style_tokens`
- `style_global`
- `timestep embedding`

主干结构：

- `12` 层 `GlyphDiTBlock`
- block 内部顺序固定为：
  1. `self-attention`
  2. `content cross-attention`
  3. `style cross-attention`
  4. `MLP`

调度策略：

- `content cross-attention`
  - 只在前 `8` 层开启
- `style cross-attention`
  - 每层开启

style 注入方式保持不变：

- `style_global`
  - 进入 AdaLN 调制
- `style_tokens`
  - 作为 style cross-attention 的 memory

每个 block 仍带有：

- `style_residual_gate`
- 初始化为 `0.1`

## 5. 训练策略

### 5.1 Stage B: Style 预训练

当前 `Stage B` 训练对象为：

- `style_encoder`
- `global_style_pool`
- `local_style_pool`
- `style_proj`
- `style_global_proj`
- `style_contrastive_head`

损失仍是 batch 内 `InfoNCE`，对比表示来自 `style_global`。

### 5.2 Stage C: Diffusion 主训练

当前默认策略是：

- 前半程冻结全部 style 分支
- 直接训练：
  - `content_encoder`
  - `content_proj`
  - `diffusion backbone`
- 到 `style_unfreeze_step` 后，再局部解冻：
  - `global_style_pool`
  - `local_style_pool`
  - `style_global_proj`
  - `style_encoder` 最后若干层
  - `style_encoder.norm`

仍保持不解冻：

- `style_proj`
- `style_contrastive_head`
- `style_encoder` 更早的 block

## 6. 结论

当前代码已经明确收敛为：

- 一条 `global style` 路径负责整体基调
- 一条 `static local style memory` 路径负责提供更丰富的候选 token
- DiT latent 在每层通过 `style cross-attention` 自己读取 style ref
- 不再保留 content-guided local routing 或其他旧分支
