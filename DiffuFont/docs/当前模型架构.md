# 当前模型架构

本文档基于当前仓库代码生成，描述的是 `DiffuFont` 里正在使用的 `content + style latent diffusion` 路径，而不是旧版 UNet / flow-matching 路径。

## 1. 总览

当前工程保留三阶段训练：

1. `Stage A: VAE 预训练`
2. `Stage B: Style 预训练`
3. `Stage C: Diffusion 主训练`

整体目标是中文字体生成。模型输入和职责如下：

- `VAE`：把 `1 x 128 x 128` 灰度字形压缩到 latent 空间 `4 x 16 x 16`
- `Content encoder`：把内容字形编码成 token 序列
- `Style encoder`：把 `R=8` 张参考字形编码成全局风格向量和局部风格 token
- `Diffusion Transformer`：在 latent 空间做条件去噪生成

当前默认训练配置来自 `train.py`：

- `device = cuda:1`
- `batch = 32`
- `style_ref_count = 8`
- `image_size = 128`
- `latent_channels = 4`
- `latent_size = 16`
- `encoder_hidden_dim = 512`
- `encoder_depth = 4`
- `dit_hidden_dim = 512`
- `dit_depth = 12`
- `local_style_tokens_per_ref = 8`
- `resolved_content_cross_attn_layers = 8`
- `style_cross_attn_every_n_layers = 2`
- `lr = 1e-4`

## 2. 数据与 batch 组织

数据由 `FontImageDataset` 提供，默认逻辑如下：

- 按字体划分训练/验证，不按字符划分
- `font_split = train`
- `font_split_seed = seed`
- `font_train_ratio = 0.9`
- 每个样本包含：
  - `content glyph`
  - `target glyph`
  - `style refs`
  - `style_ref_mask`

默认每个样本会采样 `8` 张同字体但不同字符的参考图：

- `style_img in R^(B x 8 x 1 x 128 x 128)`
- `style_ref_mask in R^(B x 8)`

只有以下场景会额外构造正样本风格对：

- `Stage B: Style 预训练`
- `Stage C` 且 `diff_lambda_ctr > 0`

此时 batch 里还会包含：

- `style_img_pos`
- `style_ref_mask_pos`

`Stage B` 默认使用 `UniqueFontBatchSampler`，尽量让一个 batch 里字体不重复，从而构造 batch 内负样本。

`Stage C` 默认不再使用 `UniqueFontBatchSampler`，而是直接走普通 shuffle dataloader，以减少主训练阶段的数据组织开销。

## 3. 模型结构

### 3.1 Glyph VAE

输入：

- `target glyph: B x 1 x 128 x 128`

编码器：

- `Conv(1 -> 32, stride=2)` + `ResBlock`
- `Conv(32 -> 64, stride=2)` + `ResBlock`
- `Conv(64 -> 128, stride=2)` + `ResBlock`
- `1x1 Conv` 输出 `mu` 和 `logvar`

latent：

- `z in R^(B x 4 x 16 x 16)`

解码器：

- `Conv(4 -> 128)`
- `ResBlock`
- 上采样到 `32 x 32`
- 上采样到 `64 x 64`
- 上采样到 `128 x 128`
- `Conv -> tanh`

### 3.2 Content Encoder

输入：

- `content glyph: B x 1 x 128 x 128`

结构：

- patch size = `8`
- 每张图切成 `16 x 16 = 256` 个 patch
- `Conv patch embed`
- 加 2D sin-cos 位置编码
- `4` 层 transformer encoder
- 不使用 `cls token`

输出：

- `content_tokens in R^(B x 256 x 512)`

如果 `encoder_hidden_dim != dit_hidden_dim`，会再通过 `content_proj` 映射到 DiT hidden dim；当前默认两者相同，因此这里实际是恒等映射。

### 3.3 Style Encoder

输入：

- `style refs: B x R x 1 x 128 x 128`
- 默认 `R = 8`

单张参考图编码：

- patch size = `8`
- 每张图切成 `256` 个 patch
- patch embed 到 `512` 维
- 加 2D 位置编码
- 加 `cls token`
- `4` 层 transformer encoder

输出：

- `local_tokens in R^(B x R x 256 x 512)`
- `global_tokens in R^(B x R x 512)`

### 3.4 多参考风格聚合

当前聚合分成两支：

- `global_style_pool`
  - 对 `R` 个全局 token 做 attention pooling
  - 输出 `global_style in R^(B x 512)`
- `local_style_pool`
  - 对每张参考图的 `256` 个局部 token 池化成 `8` 个 token
  - 然后把 `R` 张参考图拼接

最终输出：

- `style_tokens in R^(B x (8R) x 512)`
- `style_global in R^(B x 512)`
- `style_token_mask in R^(B x 8R)`

默认 `R = 8` 时：

- `style_tokens in R^(B x 64 x 512)`
- `style_token_mask in R^(B x 64)`

工程里还存在两层投影：

- `style_proj`
- `style_global_proj`

当前默认 `encoder_hidden_dim == dit_hidden_dim == 512`，所以它们是 `Identity`；如果两边维度不一致，则会自动变成线性层。

### 3.5 Style Contrastive Head

只在 `Stage B` 或 `Stage C` 显式启用对比损失时使用。

结构：

- `Linear(512 -> 512)`
- `GELU`
- `Linear(512 -> 128)`
- `L2 normalize`

输出：

- `z_style in R^(B x 128)`

## 4. Diffusion Transformer Backbone

工作空间：

- latent 空间
- 输入 latent 为 `B x 4 x 16 x 16`

条件输入：

- `content_tokens`
- `style_tokens`
- `style_global`
- `timestep embedding`

主干结构：

- `12` 层 `GlyphDiTBlock`
- block 内部顺序固定为：
  1. `self-attention`
  2. `content cross-attention`
  3. `style cross-attention`
  4. `MLP`

当前默认并不是每层都把两个 cross-attention 全打开，而是做了分层调度：

- `content cross-attention`
  - 只在前 `8` 层开启
  - 后 `4` 层关闭
- `style cross-attention`
  - 只在隔层开启
  - 默认就是第 `0, 2, 4, 6, 8, 10` 层开启

调制方式：

- `timestep embedding` 先过 `time_mlp`
- `style_global` 先过 `style_cond_proj`
- 两者相加得到 block 级条件向量 `cond`
- `cond` 通过 AdaLN 风格调制每个 block 的四个子层

### 4.1 Style 注入方式

style 条件有两条路径：

- `style_global`
  - 进入 AdaLN 调制
- `style_tokens`
  - 作为启用 style cross-attention 的那些层的 memory

内容条件同样不是每层都重复读：

- `content_tokens`
  - 主要在前 `8` 层参与 content cross-attention
  - 后几层主要负责 refinement 和 denoise

### 4.2 Style Residual Gate

这是当前版本相对旧文档的新增改动。

每个 `GlyphDiTBlock` 都有一个可学习标量：

- `style_residual_gate`
- 初始化为 `0.1`

style cross-attention 残差现在是：

`x = x + (gate_style + style_residual_gate) * style_attn_out`

其中：

- `gate_style` 来自 AdaLN 调制
- `style_residual_gate` 是每层独立学习的显式风格强度门

这意味着当前模型不会无条件把 style token 以固定强度注入到每一层，而是允许模型自行调整 style 条件的使用强度。

再加上“隔层使用 style cross-attention”的调度，当前实现里 style 条件实际上有两条不同成本的注入路径：

- 每层都存在的低成本路径：`style_global -> AdaLN`
- 隔层开启的高成本路径：`style_tokens -> style cross-attention`

## 5. 三阶段训练

### 5.1 Stage A: VAE 预训练

训练对象：

- 只训练 `VAE`

损失：

- `loss = lambda_rec * L1 + lambda_perc * perceptual + lambda_kl * KL`

当前默认：

- `vae_lambda_rec = 1.0`
- `vae_lambda_perc = 0.1`
- `vae_lambda_kl = 1e-4`

### 5.2 Stage B: Style 预训练

训练对象：

- `style_encoder`
- `global_style_pool`
- `local_style_pool`
- `style_proj`
- `style_global_proj`
- `style_contrastive_head`

损失：

- batch 内 `InfoNCE`

batch 组织：

- 默认启用 `UniqueFontBatchSampler`
- 一个 batch 尽量由不同字体组成
- `style_img` 和 `style_img_pos` 形成正样本对

### 5.3 Stage C: Diffusion 主训练

输入：

- `content glyph`
- `target glyph`
- `R` 张 style refs

训练对象主干：

- `content_encoder`
- `content_proj`
- `diffusion backbone`

默认冻结对象：

- `VAE`
- style 分支前半程全部冻结

默认主损失：

- `loss = diffusion_mse`

也就是当前默认值为：

- `diff_lambda_diff = 1.0`
- `diff_lambda_rec = 0.0`
- `diff_lambda_perc = 0.0`
- `diff_lambda_kl = 0.0`
- `diff_lambda_ctr = 0.0`

虽然代码仍然保留了 `recon`、`perceptual`、`KL`、`contrastive` 的开关，但默认主训练只优化噪声预测 MSE。

当前默认的主训练效率策略还包括：

- 不默认启用 `--train-style-jointly`
- 不让 `UniqueFontBatchSampler` 参与 Stage C
- 把高成本的 cross-attention 频率压低
- 把 `style_tokens` 长度从 `128` 压到 `64`

## 6. Stage C 的 style 冻结与解冻策略

这是当前实现里最关键的训练策略改动。

### 6.1 默认策略

如果使用默认配置：

- `train_style_jointly = False`
- `style_unfreeze_step = 5000`
- `style_unfreeze_last_encoder_blocks = 1`

那么真实行为是：

1. 训练开始时，style 分支全部冻结
2. 当训练步数达到 `5000` 时，自动开启局部 style finetune

### 6.2 自动解冻的模块

当前代码会解冻：

- `global_style_pool`
- `local_style_pool`
- `style_global_proj`
- `style_encoder` 最后 `N` 个 block
- `style_encoder.norm`

默认 `N = 1`。

当前代码不会在这个阶段自动解冻：

- `style_proj`
- `style_contrastive_head`
- `style_encoder` 更早的 block

### 6.3 学习率

Stage C 里优化器分为两组：

- 主干参数：`lr`
- style 局部 finetune 参数：`lr * style_lr_scale`

当前默认：

- `style_lr_scale = 0.1`

也就是 style 局部解冻部分默认使用主干学习率的 `0.1x`。

### 6.4 特殊情况

如果显式传入：

- `--train-style-jointly`

那么 style 分支从一开始就参与训练，不再走“前半程冻结、后半程局部解冻”的默认策略。

## 7. 采样与推理

当前 `DiffusionTrainer` 里带有 `ddim_sample`：

- 默认推理步数可配
- 从高斯噪声开始
- 在 latent 空间逐步去噪
- 最后通过 `VAE decoder` 解码回 `1 x 128 x 128` 字形图

训练过程中会定期保存 sample 图，默认会混合：

- 训练集 seen fonts
- 验证集 unseen fonts

## 8. 当前版本的关键信息总结

当前版本相对早期设计，最重要的几点是：

- 模型主干已经固定为 `latent DiT`，不是 UNet
- 多参考默认固定为 `R = 8`
- 每张参考图默认只保留 `8` 个局部 style token，所以总 style memory 长度默认是 `64`
- `Stage C` 默认只用 `diffusion_mse`
- `Stage C` 不再默认把整个 style 分支完全冻结到底
- style 条件在每层都有显式 `style_residual_gate`
- style cross-attention 默认隔层使用
- content cross-attention 默认只保留前 `8` 层
- style 后半程只局部解冻聚合层、`style_global_proj`、以及 `style_encoder` 最后若干层

如果后续代码继续改动，优先以 `train.py`、`models/source_part_ref_dit.py`、`models/diffusion_transformer_backbone.py`、`models/model.py` 为准。
