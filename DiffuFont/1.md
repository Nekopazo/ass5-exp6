1. **把单一 style 向量改成 3 个 style token**
    * 原来：`style_vec`
    * 改成：`t_low, t_mid, t_high`
2. **Style Encoder 输出改造**
    * 保留原 CNN 提特征
    * 去掉“最后只输出一个全局 style 向量”的做法
    * 改成用 `TokenAggregator / Cross-Attn` 从 reference 特征里聚合出 3 个 token
3. **预训练输出改造**
    * 不再保存单个 pooled style
    * 改成保存整组 token：`[t_low, t_mid, t_high]`
4. **UNet 注入方式改造为分层硬分配**
    * `mid <- t_low`
    * `up16 <- t_mid`
    * `up32 <- t_mid`
    * `up64 <- t_high`
5. **每个注入层加独立 inject head**
    * 新增：
        * `H_mid(t_low)`
        * `H_16(t_mid)`
        * `H_32(t_mid)`
        * `H_64(t_high)`
    * 不要所有层共用一个 style head
6. **不要再把 3 个 token 池化回一个向量再注入**
    * 禁止 `mean([t_low, t_mid, t_high]) -> 所有层`
    * 必须保持 token 分开走不同层
7. **预训练 loss 改造**
    * 保留字体级一致性：`font-nce`
    * 加槽位一致性：`slot-cons`
    * 可选再加轻量去冗余：`token-div`
    * 加频段 proxy loss：`low/mid/high` 各自的小任务
8. **主训练 loss 改造**
    * 主损失仍然是 diffusion loss
    * 额外保留轻量辅助项：
        * `slot-cons`
        * `token-div`
    * 如果需要更稳，再加 `font-nce` 小权重辅助
9. **训练策略改造**
    * diffusion 从头训练时：
        * 先冻结 style encoder 的 CNN backbone
        * 训练 token aggregator、3 个 token、4 个 inject heads、UNet
    * 中后期再小学习率解冻 CNN 高层
10. **cross-attention 使用策略改造**

* 不要让所有层自由读取全部 token
* 第一版直接硬分配


11. **命名建议改造**

* 统一改叫：
    * `t_low`
    * `t_mid`
    * `t_high`

预训练阶段：加 proxy loss

作用：尽早把 t_low / t_mid / t_high 拉开

这是最有价值的阶段

主训练阶段：也保留，但要降权

作用：防止 diffusion 主任务把 3 个 token 又训塌回去

不能和预训练同权重，否则会压主任务
